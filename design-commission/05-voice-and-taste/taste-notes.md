# Cem's recorded design taste — from org memory

Collected from decisions, feedback, and artifact history. These are recorded
preferences, not guesses.

## Palette

- **Rejected explicitly**: gold-parchment / vellum-brass pairings ("vellum/brass"
  artifact palette family). Do not build the identity on aged-paper gold.
- **Preferred**: the "sovereign" palette family; the live Meridian system
  (paper `#eceef2`, ink `#15212e`, prussian blue, with brass/teal/rose as
  *accent roles*, never the base) is accepted and working.
- Always ship a theme/palette picker on artifacts (the ❖ picker convention) —
  for web surfaces this translates to first-class dark mode (`meridian-nocturne`
  token set exists).

## Composition

- **One evolving visual** beats many small boxes. In decks: a single visual that
  develops across slides, prose in the notes. In pages: a hero that carries one
  strong idea, not three equal cards.
- Structure over decoration: forks, ledgers, staged flows — make the *shape* of
  the content visible.
- Wall-of-prose is failure; so is fragment-soup. Statements with visible
  hierarchy.

## Language

- Plain language over neologisms — no new coinages in UI labels or designs.
  Established org terms only: emissary, directory (not "shelf" — retired),
  feed, stars, pull, harness.
- Metaphor lives in visuals, not labels.
- External copy follows the egregore-voice register (see egregore-voice.md):
  manuscriptic, architecturally precise, philosophically grounded, zero
  AI-slop cadence.

## Quality bar

- The internal reference for "flagship" is the Meridian-rendered decision
  surfaces and living papers (scrolls) — self-contained HTML artifacts with
  explanatory faces, componentized sections, and deliberate typography.
- If a surface would look at home in a generic SaaS template, it fails.
- The client reacted to the current landing page with "is this a fucking joke?"
  — calibrate accordingly.
