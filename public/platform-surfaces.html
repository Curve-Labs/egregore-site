§Emissary Platform · Surface Mapfor Cem · MMXXVI

Every surface, and what's missing.

An emissary is a portable, runnable task you hand to another person — readable as a page, executable by their agent. The platform is where those live at a permanent address, get discovered, and accumulate proof they work. The publishing/running substrate is real. The product Cem is describing is largely the web account + engagement layer on top of it.

02The Critical Path

Everything hangs off one decision and one build

1 · Decide the web-session model

Magic-link sign-in vs. token-as-credential. Nothing logged-in can be built until this is chosen. (See §03.)

2 · Web signup + sign-in

Turn the existing /register + /verify API into real web surfaces with a session.

3 · Profile + Settings

Wire up bio/links/featured (already in schema). Cheapest high-visibility win once sessions exist.

4 · Web starring

On Shelf + Emissary view. Makes the star metric real — it's all zeros today because there's no signed-in human to star.

5 · Follow → Feed → Notifications

The retention loop. Largest net-new build; do last.

03The Open Decision

The one that unblocks the rest

How does a human get a web session?

Without this there is no logged-in web experience — no starring, no settings, no "my emissaries." Pick before building anything in the engagement or account layers.

Recommended A · Magic-link sign-in
+Reuses the /verify mechanism that already works — no new credential type.
+Passwordless; matches the email-first identity already minted at register.
−Email round-trip per new device; needs session cookie + expiry handling.
B · Registration token as web credential
+Zero new infra — the token already exists from /register.
−A long-lived bearer token in the browser is a worse security posture.
−Conflates the agent (MCP) credential with the human web session.
  • Handle = identity? Is @handle the durable identity, or a vanity layer over email? Shapes signup + settings.
  • Stars vs. carries as the headline? Carries are honest (agents running things); stars need humans + sessions.
  • How much of compose comes to the web? Recommend: composition stays agent-conducted; management comes to web.
  • Follow people, topics, or both? Determines the feed's shape.
  • Public vs. invite-only? The shelf is gated today (team-gated); signup design depends on when that comes down.
